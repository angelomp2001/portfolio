# Sprint 12 — Car Price Prediction


## Business Case
A client wants to offer customers an automated estimate of what their car could sell for.  
**Task:** Predict `Price` using structured car listing data.


## Project Summary
This project trains and evaluates several regression models on a dataset of ~354,000 car listings. The best model is selected based on RMSE and training/prediction time. The best model is then served via a REST API for inference on new car listings.

## How to train model
```bash
python main.py
```


## How to run inference via API
```bash
uvicorn app:app --reload
```
Send POST requests to `http://127.0.0.1:8000/predict` with a JSON body containing car feature fields.


## Train Pipeline Overview
├── Load — Read `data.csv`, sample 10,000 rows (optional)
├── Clean — Review each column, fix/remove bad values, feature engineer
├── Split — Split data via cross-validation, and apply the following:
│   ├── Encode  — convert text values into numerical values
│   ├── Scale — standardize numerical values
│   ├── Train — Fit models
│   ├── Evaluate — Compare metric, train time, prediction time across models
│   ├── Tune — Hyperparameter search on best model
└── Save — Save best model  


## API Overview
├── Load — Load best model
